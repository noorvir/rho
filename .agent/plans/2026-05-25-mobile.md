# Current Work: Mobile Shell POC

## Goal

Add a minimal native iOS shell that can run on a phone/simulator. The POC should prove a blank native surface with a global chat affordance in the top-right.

## Branch

`mobile`

## Package Shape

Use `packages/mobile` for the native iOS app. Keep it independent from `@rho/cli` and the TypeScript package graph so CLI installs do not pull mobile dependencies.

## Current Direction

The POC moved from Expo/React Native to SwiftUI because the desired shell is iOS-first and benefits from native safe areas, sheets, keyboard behavior, and system theming. Keep the app intentionally bare: white background, native top bar, `Chat` button, native chat sheet.

## App Structure

- `packages/mobile/RhoMobile.xcodeproj`: minimal Xcode project.
- `packages/mobile/RhoMobile/RhoMobileApp.swift`: SwiftUI app entrypoint.
- `packages/mobile/RhoMobile/ContentView.swift`: blank shell with top-right chat button.
- `packages/mobile/RhoMobile/ChatView.swift`: native chat sheet with local echo behavior.

## UI Behavior

The root view is blank and light-mode by default. Pressing `Chat` presents a native SwiftUI sheet. The sheet contains a chat title, scrollable message list, message input, and send button. Sending a message appends the user message and a local `echo: ...` assistant response.

## Non-goals

- Android support, Expo, React Native, auth, backend clients, persistence, push notifications, or production navigation polish.
- Wiring chat to `@rho/ai`, coding-agent tasks, or channel backends.
- Sharing mobile dependencies with `@rho/cli`.
