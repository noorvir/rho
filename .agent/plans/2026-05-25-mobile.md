# Current Work: Mobile Shell POC

## Goal

Build a minimal native iOS SwiftUI shell for `rho` with reusable bottom bars, a chat sheet surface, and room to add real app screens next.

## Branch

`mobile`

## Current State

The mobile POC is a standalone SwiftUI iOS app. The current visible shell is intentionally blank except for the bottom navigation/action UI. Demo content has been removed.

## UI Direction

- Light mode by default.
- Use safe-area-aware SwiftUI layout primitives instead of absolute positioning, negative offsets, or screenshot-specific coordinate tweaks.
- Keep controls compact and square-ish: small corner radii, subtle thin borders, light shadows.
- Keep popovers and secondary surfaces no wider than their invoking bottom-bar group unless intentionally promoted to a sheet.
- Avoid oversized icons, fonts, and padding; match the bottom bar's compact visual density.
- Keep placeholder surfaces blank until real app content exists.

## Current Interface Shape

Pseudocode:

```swift
AppShell(bottomBar: .root(openChat: ...))
AppShell(bottomBar: .pageActions(openChat: ...))
ChatSheet(input: ...)
```

The root bottom bar has a grouped navigation control plus a standalone chat bubble button. Page-specific screens can use a compact action/comment/send bar instead.

## Non-goals

- Android support, backend clients, persistence, push notifications, or production navigation polish.
- Wiring chat to AI/channel backends yet.
- Reintroducing demo issue/task content.

## Validation

- TypeScript workspace checks pass.
- Native iOS app builds, installs, and launches in the simulator.
- Visual checks confirm the compact bottom shell layout.
