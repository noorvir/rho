Channels are adapters. Turns are the product boundary.

A comparative study of OpenClaw, Hermes Agent, ZeroClaw, and Nanobot: how they route chat platforms, stream output, authorize users, cancel work, and move media/voice through agent systems.

OpenClaw

Full gateway + harness boundary

  • Host prepares turns before harness execution.
  • Live previews/finalization are platform-aware.
  • Strong media/voice/plugin surfaces.
Hermes

Gateway stream consumer

  • Sync token callbacks bridge into async adapters.
  • Allowlists and DM pairing gate sessions.
  • Voice is adapter capability.
ZeroClaw

Trait-based channel runtime

  • Draft lifecycle is the streaming interface.
  • Explicit cancel/finalize hooks.
  • Media markers normalized centrally.
Nanobot

Small bus + adapters

  • Minimal Inbound/Outbound message bus.
  • Stream deltas are metadata.
  • WhatsApp lives behind hardened local bridge.

Convergence matrix

ConcernWhat others doRho stance
StreamingOpenClaw live previews, Hermes stream consumer, ZeroClaw drafts, Nanobot delta metadata.Reuse Pi `AgentEvent`s internally; channels decide how to render.
AuthSeparate platform credentials from user allowlists/pairing and admin command tiers.Three layers: connector auth, user authorization, capability authorization.
CancelAbort controllers/tokens, `/stop`, session cancel, disconnect reasons.AgentRunRegistry owns AbortSignal and cancellation reason before ack.
MediaClaim-check stores or local path markers; safe roots and MIME checks.Typed content parts backed by a media store; markers only at provider/channel edges.
VoiceEither media+transcript or separate realtime audio stream sessions.Start as audio attachment/transcript; realtime voice gets its own session API later.
ExtensibilityPlugins/factories/bridges; WhatsApp often separated into local bridge.File-backed `ChannelDefinition` + `ChannelFactory`, loaded by `/reload`.

Interfaces to add next

  • AgentRun for persisted/cancellable execution.
  • Wrapped Pi AgentEvents for streaming.
  • AgentRunRegistry for cancellation and active session ownership.
  • ChannelFactory for file-backed reloadable connectors.
  • MediaStore before adding image/audio uploads.

Anti-patterns to avoid

  • Do not let each channel call the agent with its own shape.
  • Do not expose raw LLM chunks as the extension protocol.
  • Do not mix platform tokens with user authorization.
  • Do not implement media parsing separately per route.
  • Do not treat realtime voice as just another text message.

  source: ChannelSource;

  messages: AgentMessage[];
  abortReason?: CancelReason;
};

type AgentRunEvent = {
  runId: string;
  event: AgentEvent;
